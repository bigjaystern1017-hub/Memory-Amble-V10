## 1. MOBILE HERO FIX (highest priority)

The hero section currently uses a two-column layout on desktop: headline on the left, the purple palace illustration on the right. On desktop this looks great. On mobile it's broken — the illustration gets pushed far below the fold and the user sees a wall of text before ever seeing the image.

Fix the mobile layout so the illustration is visible above the fold on iPhone-sized screens (375px wide). Here's the approach:

**On mobile (below `md:` breakpoint):**
- Show the illustration FIRST, spanning full width, at a max height of roughly 240-260px, with `object-cover` and `object-position: top` so we see the top portion of the palace (the people walking, the arches, the sunset). Wrap it in a div with `overflow-hidden` and `rounded-b-2xl` so it feels intentional, not cropped.
- Below the illustration: the pill badge, headline, body text, and CTA — with TIGHT spacing. Reduce all vertical gaps (`space-y`, `py`, `mb`, `gap`) by about 40% compared to the current mobile layout. The goal is: illustration + headline + CTA all visible without scrolling on a standard iPhone screen.
- The three stat pills (10 min/day, Science-backed, Made for life) should sit in a compact horizontal row below the CTA on mobile.

**On desktop (`md:` and up):**
- Keep the current two-column layout exactly as it is. Headline/CTA on the left, full illustration on the right. Don't change anything about the desktop hero.

Use Tailwind responsive classes (`hidden md:block`, `md:hidden`, `md:flex-row`, etc.) to serve different layouts. You can render the illustration image twice in the JSX (once for mobile, once for desktop) and hide/show with responsive classes — that's fine.
