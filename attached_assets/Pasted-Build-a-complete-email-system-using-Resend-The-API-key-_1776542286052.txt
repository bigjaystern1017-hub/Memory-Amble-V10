3. TRIGGER THE WELCOME EMAIL ON SIGNUP

In server/routes.ts (or wherever user signup/registration is handled), find the point where a new user is created after signing up. This might be in a webhook handler or after the first progress save. 

The simplest approach: in the POST /api/progress endpoint, when a NEW user record is created (the insert path, not the update path), call sendWelcomeEmail. Find the block that does:

const [created] = await db.insert(userProgress).values(...)

After that insert, add:

import { sendWelcomeEmail } from './email';

// Get user email from the auth token
try {
  const userEmail = req.user?.email;
  if (userEmail && userName) {
    sendWelcomeEmail(userEmail, 'there');
  }
} catch (e) {
  console.error('Welcome email failed:', e);
}

Note: we might not have the userName at this point — if not, just use 'there' as the name ("Hi there,"). The important thing is the email sends. Don't let it block the API response — fire and forget.

Actually, a better place to get both email and name: look at the POST /api/sessions endpoint. When the FIRST session is saved (you can check if this is the user's first by checking if any prior sessions exist), send the welcome email. The session save has access to the full session data including userName via the request body.

Add this logic to POST /api/sessions, right after the successful insert:

// Send welcome email on first session
try {
  const existingSessions = await db.select().from(sessionHistory).where(eq(sessionHistory.userId, userId));
  if (existingSessions.length === 1) {
    // This was their first session
    const { sendWelcomeEmail } = await import('./email');
    const userEmail = req.user?.email;
    if (userEmail) {
      sendWelcomeEmail(userEmail, req.body.placeName ? 'friend' : 'there');
    }
  }
} catch (e) {
  console.error('Welcome email failed:', e);
}

4. CREATE THE DAILY REMINDER ENDPOINT

Add a new endpoint to server/routes.ts:

app.get("/api/send-reminders", async (req, res) => {
  // Simple auth: check for a secret key so this can't be called by random people
  const authKey = req.headers['x-reminder-key'];
  if (authKey !== process.env.REMINDER_SECRET) {
    return res.status(401).json({ error: "Unauthorized" });
  }

  try {
    const { sendReminderEmail } = await import('./email');
    
    // Get all users with their progress
    const allProgress = await db.select().from(userProgress);
    
    let sent = 0;
    let skipped = 0;
    
    for (const progress of allProgress) {
      // Skip users who already played today
      const today = new Date().toISOString().slice(0, 10);
      const lastLogin = progress.lastLogin ? new Date(progress.lastLogin).toISOString().slice(0, 10) : null;
      
      if (lastLogin === today) {
        skipped++;
        continue;
      }
      
      // Skip users with no sessions yet (they haven't started)
      if (progress.dayCount === 0) {
        skipped++;
        continue;
      }
      
      // Get user email — we need to look this up from the auth system
      // Since we're using Supabase auth, we need the user's email
      // For now, skip if we can't determine email
      try {
        // Try to get email from Supabase admin
        const { createClient } = await import('@supabase/supabase-js');
        const supabaseAdmin = createClient(
          process.env.SUPABASE_URL!,
          process.env.SUPABASE_SERVICE_ROLE_KEY!
        );
        const { data: userData } = await supabaseAdmin.auth.admin.getUser(progress.userId);
        const email = userData?.user?.email;
        
        if (email) {
          await sendReminderEmail(email, 'friend', progress.dayCount, progress.streak);
          sent++;
        } else {
          skipped++;
        }
      } catch (e) {
        console.error(`Failed to send reminder to user ${progress.userId}:`, e);
        skipped++;
      }
    }
    
    res.json({ sent, skipped, total: allProgress.length });
  } catch (error) {
    console.error("Reminder send failed:", error);
    res.status(500).json({ error: "Failed to send reminders" });
  }
});

5. Add REMINDER_SECRET to Replit Secrets — just make up any random string like "timbuk-reminders-2026" and add it as a secret. This prevents random people from triggering reminder emails.

6. To actually send daily reminders, you would set up a cron job or external scheduler to call:

GET https://memoryamble.com/api/send-reminders
Header: x-reminder-key: [your REMINDER_SECRET value]

For now, you can trigger this manually by visiting that URL with the right header (use a tool like curl or Postman). Later you can set up a free cron service like cron-job.org to call it every morning at 9am.
